如何开始？

# 快速开始

npx

# 初始化，并选择 2.x
npx @vuepress-reco/theme-cli init

npm

# 初始化，并选择 2.x
npm install @vuepress-reco/theme-cli@1.0.7 -g
theme-cli init

yarn

# 初始化，并选择 2.x
yarn global add @vuepress-reco/theme-cli@1.0.7
theme-cli init

# 安装与配置

reco 2.xopen in new window

依次执行以下命令

theme-cli是一个脚手架，使用它来初始化可以免去很多配置 初始化完成后，先不要着急启动，做一些基本配置

nvm list
nvm use 16.14.0
npm install @vuepress-reco/theme-cli@1.0.7 -g
theme-cli init <blog>
cd blog
npm install

初始化后的项目结构

  • 版本 reco2.0.0-beta.48

基本配置

  1. 删除blogs和docs文件夹下的初始md文件
  2. 更改temp临时文件目录的位置，修改package.json运行脚本为"dev": "vuepress dev . --temp .temp"
  3. 在主配置文件config.ts中做一些配置
  4. 新建.vuepress/styles/index.css，在其中书写自定义样式
// .vuepress/config.ts
export default defineUserConfig({
  title: '星期天',
  description: '个人知识库',
  dest: './dist',  // #build部署文件夹的位置
  port: 8081,      // #运行端口号
  head: [          // #inject on <head></head>
    ['link', { rel: 'icon', href: 'https://mysource-hexo.wyun521.top/img/cover/logo.png' }],
  ],
  theme: recoTheme({    // #主题配置
    // #添加的内容
    catalogTitle: '文章目录',
    authorAvatar: 'https://mysource-hexo.wyun521.top/img/avatar/7.jpg',

    // series 为原 sidebar
    series: {
      '/docs/theme-reco/': [
        {
          text: 'module one',
          children: ['home', 'theme']
        },
        {
          text: 'module two',
          children: ['api', 'plugin']
        }
      ]
    },
    navbar:
    [
      { text: 'Home', link: '/' },
      { text: 'Categories', link: '/categories/reco/1/' },
      { text: 'Tags', link: '/tags/tag1/1/' },
      { text: 'Docs',
        children: [
          { text: 'vuepress-reco', link: '/docs/theme-reco/theme' },
          { text: 'vuepress-theme-reco', link: '/blogs/other/guide' }
        ]
      },
    ],

  })
})

reco 主题实现了博客的功能，但是 vuepress 没有办法区分博客或是文档，导致文档也会出现在博客区域，所以主题对博客和文档的存放位置进行了约束，博客文章请放在 /blogs 文件夹中（强制），文档请放在 /docs 中（不强制）。

# 主页配置

reco 2.x已经支持首页的模块化定制，目前内置的首页模块有 Banner、BannerBrand、Blog、MdContent、Footer，在首页 md 文件的 frontmatter 中指定要展示的内容模块及其展示顺序

# 运行

npm run dev
